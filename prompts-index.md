# VibeUI Prompts Index

Complete reference of all 92 prompts. Each prompt ends with the style suffix:
> *Match the visual style, colors, typography, and overall aesthetic of the UI shown in my screenshot.*

---

## 1. Auth Forms (6)

### auth-1 | Split-screen with visual
Create an auth form as a split-screen layout. Place the form fields on one side and a visual panel on the other (product screenshot, illustration, or branded graphic area). Include email, password, and a primary CTA.

### auth-2 | Centered card on background
Create an auth form as a centered floating card on a full-page background. Include heading, email, password, primary CTA, and a link to switch to sign up.

### auth-3 | Minimal single-column
Create an auth form as a minimal single-column layout with no card container — just the logo, heading, form fields, and CTA stacked cleanly down the page.

### auth-4 | Social-first stack
Create an auth form with social login buttons (Google, GitHub, Apple) as the primary option at the top, and email/password collapsed or secondary below a divider that says "or continue with email".

### auth-5 | Magic link only
Create an auth form as a magic-link-only flow. A single email input and one primary CTA button — no password field. Include a short line of helper text explaining a login link will be sent to their email.

### auth-6 | Multi-step wizard
Create an auth form as a multi-step wizard with a progress indicator at the top. Split signup across 2–3 logical steps (e.g. email → password → profile details) with back/next navigation.

## 2. Pricing (8)

### pr-1 | Classic 3-tier cards
Create a pricing section with three side-by-side tier cards (e.g. Starter, Pro, Business) with the middle tier visually highlighted as the recommended option. Include plan names, prices, feature lists, and CTAs.

### pr-2 | Monthly/annual toggle
Create a pricing section with a monthly/annual billing toggle at the top and a "save X%" badge on the annual option. Prices should visibly update based on the toggle.

### pr-3 | Comparison table
Create a pricing section as a detailed comparison table with plans as columns and features as rows, using checkmarks and dashes to show what's included in each tier.

### pr-4 | Single-tier hero
Create a pricing section as a single-tier hero layout — one large centered card with the price, a full feature list, and a prominent CTA.

### pr-5 | Usage slider
Create a pricing section with an interactive usage slider at the top (e.g. users, requests, or seats) that dynamically updates the displayed price below.

### pr-6 | Credit pack grid
Create a pricing section as a grid of credit pack tiles, each showing the credit amount, price, and a "best value" or "most popular" badge on one option.

### pr-7 | Free vs Pro split
Create a pricing section as a two-column split between a Free tier and a Pro tier, with strong visual contrast between the two so the upgrade path is obvious.

### pr-8 | Stacked mobile-first cards
Create a pricing section as full-width stacked cards, one per tier. No horizontal comparison — each card stands on its own with its full feature list visible.
